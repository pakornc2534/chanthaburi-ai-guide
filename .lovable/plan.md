
# TripChan — AI แนะนำเที่ยวจันทบุรี

แอพเว็บที่ใช้ AI ช่วยแนะนำสถานที่ท่องเที่ยวในจังหวัดจันทบุรี รองรับทั้งการแชทถาม-ตอบและการสร้างแผนเดินทางอัตโนมัติ พร้อมฐานข้อมูลสถานที่ที่ดูแลไว้แล้ว แผนที่ และระบบบันทึกรายการโปรด/แชร์

## หน้าหลักของแอพ

**1. หน้าแรก (/)** 
- Hero แนะนำแอพพร้อมภาพจันทบุรี (ทะเล/ภูเขา/ผลไม้)
- สวิตช์ภาษา ไทย/EN มุมขวาบน (จำค่าใน localStorage)
- ปุ่ม CTA หลัก 2 อัน: "แชทกับ AI" และ "สร้างแผนเที่ยว"
- Section สถานที่แนะนำ (Top picks) ดึงจาก DB
- Section หมวดหมู่: ทะเล, ภูเขา/น้ำตก, วัด/ประวัติศาสตร์, คาเฟ่, ตลาด, ผลไม้/สวน

**2. หน้ารวมสถานที่ (/places)**
- กริดการ์ดสถานที่ พร้อมรูป ชื่อ หมวด คะแนน คำอธิบายสั้น
- ฟิลเตอร์: หมวดหมู่, อำเภอ, ฟรี/มีค่าเข้า
- ค้นหาด้วยคีย์เวิร์ด
- ปุ่มหัวใจบันทึกโปรด

**3. หน้ารายละเอียดสถานที่ (/places/$id)**
- รูป, คำอธิบายเต็ม (ไทย/EN), เวลาเปิด, ค่าเข้า, ที่อยู่, เบอร์
- แผนที่แสดงตำแหน่ง (Leaflet + OpenStreetMap, ฟรี ไม่ต้องใช้ API key)
- ปุ่ม "เปิดใน Google Maps", "บันทึกโปรด", "ถาม AI เกี่ยวกับที่นี่"
- สถานที่ใกล้เคียง

**4. หน้าแชท AI (/chat)**
- อินเทอร์เฟซแชทแบบสตรีมมิ่ง รองรับ markdown
- AI ตอบเป็นภาษาที่ผู้ใช้เลือก ใช้ context จากฐานข้อมูลสถานที่ในจันทบุรี
- ตัวอย่างคำถาม (chips): "ที่เที่ยวธรรมชาติ 1 วัน", "คาเฟ่ริมทะเล", "ของกินห้ามพลาด"
- เก็บประวัติการสนทนา (เฉพาะ session)

**5. หน้าสร้างแผนเที่ยว (/planner)**
- ฟอร์ม: จำนวนวัน (1-5), งบ (ประหยัด/ปานกลาง/พรีเมียม), ความสนใจ (multi-select: ทะเล/ภูเขา/วัด/อาหาร/คาเฟ่/ผลไม้), เดินทางแบบ (ครอบครัว/คู่/เพื่อน/คนเดียว), เดือนที่ไป
- กดสร้าง → AI ส่งคืนแผนแบบมีโครงสร้าง (JSON) แสดงเป็น timeline แต่ละวัน เช้า/กลางวัน/บ่าย/เย็น พร้อมลิงก์ไปหน้ารายละเอียดสถานที่
- ปุ่ม "บันทึกแผน", "แชร์แผน" (สร้างลิงก์สาธารณะ), "ปรับแผน" (ส่งกลับให้ AI แก้)

**6. หน้ารายการโปรด (/favorites)**
- สถานที่ที่บันทึกไว้ + แผนเที่ยวที่บันทึก
- ใช้ได้แบบไม่ล็อกอิน (เก็บ localStorage) หรือล็อกอินเพื่อ sync ข้ามอุปกรณ์

**7. หน้าแชร์แผน (/trip/$shareId)**
- เปิดดูแผนสาธารณะที่คนอื่นแชร์มา (read-only)

## ฟีเจอร์ AI

- **AI Chat**: สตรีมมิ่งคำตอบทีละ token ใช้ Lovable AI Gateway (โมเดลเริ่มต้น google/gemini-3-flash-preview) System prompt ฝังบริบทว่าเป็นไกด์จันทบุรี และให้แนะนำเฉพาะที่อยู่ในฐานข้อมูล
- **Trip Planner**: เรียก AI แบบ structured output (tool calling) ส่งคืน JSON schema {days: [{day, items: [{time, placeId, activity, note}]}]} เพื่อ render เป็น UI ได้เป๊ะ
- **Place Q&A**: ถาม AI เกี่ยวกับสถานที่หนึ่งๆ โดยฝังข้อมูลของสถานที่นั้นเป็น context
- ทุกการเรียก AI ทำผ่าน server function — ไม่เปิด API key ฝั่ง client

## ฐานข้อมูล (Lovable Cloud)

- `places` — id, name_th, name_en, category, district, description_th, description_en, image_url, lat, lng, opening_hours, price_info, address, phone, tags
- `favorites` — user_id (nullable), place_id, created_at (สำหรับผู้ที่ล็อกอิน)
- `trips` — id, share_id (unique), user_id (nullable), title, params (jsonb: วัน/งบ/ความสนใจ), plan (jsonb: แผนจาก AI), is_public, created_at
- ผม seed ข้อมูลสถานที่ยอดนิยมในจันทบุรี ~25-30 แห่ง (ทะเล: หาดเจ้าหลาว/แหลมเสด็จ/อ่าวคุ้งกระเบน, ภูเขา/น้ำตก: น้ำตกพลิ้ว/เขาคิชฌกูฏ/น้ำตกกระทิง, วัด/ประวัติ: อาสนวิหารพระนางมารี/ชุมชนริมน้ำจันทบูร/ค่ายตากสิน, คาเฟ่ริมทะเล, ตลาดพลอย, สวนผลไม้)
- รูปสถานที่ใช้ Unsplash/แหล่งภาพฟรี

## ระบบล็อกอิน (เลือกใช้ได้)

- ผู้ใช้ใช้แอพได้ทันทีโดยไม่ต้องล็อกอิน (favorites/แผน เก็บใน localStorage)
- มีปุ่ม "เข้าสู่ระบบ" เพื่อ sync ข้ามอุปกรณ์ (อีเมล + รหัสผ่าน ผ่าน Lovable Cloud Auth)
- RLS เปิดใน trips/favorites ให้แก้ได้แค่เจ้าของ; trips ที่ is_public=true อ่านได้ทุกคน

## ภาษา ไทย/อังกฤษ

- Toggle ใน header, จำค่าใน localStorage
- ใช้ context provider ง่ายๆ + dictionary object สำหรับ UI strings
- คอนเทนต์สถานที่มีคอลัมน์ _th และ _en แยก
- ส่งภาษาที่เลือกไปให้ AI ใน system prompt เพื่อให้ตอบภาษานั้น

## ดีไซน์

- โทน: ฟ้าทะเล + เขียวธรรมชาติ + ส้มผลไม้ (จันทบุรี = ทะเล/ภูเขา/ผลไม้/พลอย) บนพื้นขาวสะอาด รองรับ dark mode
- ฟอนต์: Prompt (ไทย) + Inter (อังกฤษ)
- การ์ดมุมมน เงาเบา รูปสวย เน้นภาพถ่าย
- มือถือเป็นหลัก: bottom nav 4 แท็บ (Home / Places / Chat / Trips)

## รายละเอียดทางเทคนิค

- เปิด Lovable Cloud (DB + Auth) และ Lovable AI Gateway
- Server functions สำหรับ: chat (สตรีม SSE), generate-trip (structured output), get-places, save-trip
- Trip planner ใช้ tool calling เพื่อบังคับ schema ของแผน
- แผนที่: ใช้ react-leaflet + OpenStreetMap tiles (ไม่มีค่าใช้จ่าย ไม่ต้องใส่ API key)
- แชร์แผน: สร้าง share_id แบบสุ่ม, route /trip/$shareId อ่านจาก DB ตรงๆ
- จัดการรายการโปรดแบบ optimistic update; ถ้าไม่ล็อกอินใช้ localStorage, ถ้าล็อกอินอยู่ก็ sync เข้า DB

หลังกด Implement ผมจะเริ่มจาก: ตั้งค่า Cloud + schema + seed สถานที่ → หน้าแรก/Places/รายละเอียด → AI Chat → Trip Planner → Favorites/Share → ขัดเกลาดีไซน์และภาษา
